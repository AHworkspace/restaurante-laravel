1. MAPA RAPIDO PARA CAMBIOS FRECUENTES
======================================

Dashboard y menu visible:
- Vista: sistema-web\resources\views\home.blade.php
- Controlador: sistema-web\app\Http\Controllers\HomeController.php
- Modelo y horario: sistema-web\app\Models\MenuDia.php
- Gestion: sistema-web\app\Http\Controllers\MenuDiaController.php
- Modal de consumo: sistema-web\resources\views\consumos\modal-registro-rapido.blade.php

Consumos y ventas desde el dashboard:
- Logica: sistema-web\app\Http\Controllers\ConsumoController.php
- Modelos: sistema-web\app\Models\Consumo.php y Venta.php
- Pantallas: sistema-web\resources\views\consumos\
- Rutas: sistema-web\routes\web.php

Pagos y saldos:
- Logica: sistema-web\app\Http\Controllers\PagoController.php
- Modelos: sistema-web\app\Models\Pago.php, Consumo.php y Consumidor.php
- Pantallas: sistema-web\resources\views\pagos\
- Cuenta del cliente: sistema-web\resources\views\consumidores\show.blade.php

Compras e inventario:
- Compras: sistema-web\app\Http\Controllers\CompraController.php
- Movimientos: sistema-web\app\Http\Controllers\MovimientoInventarioController.php
- Modelos: Compra.php, CompraLinea.php, CompraProveedorPago.php,
  MovimientoInventario.php e Insumo.php

Usuarios, roles y acceso:
- Rutas: sistema-web\routes\web.php
- Autenticacion: sistema-web\config\auth.php
- Roles: sistema-web\config\permission.php
- Navegacion por rol: sistema-web\resources\views\layouts\navigation.blade.php

Historial general:
- Escritura: sistema-web\app\Helpers\HistorialHelper.php
- Consulta: sistema-web\app\Http\Controllers\HistorialController.php
- Pantalla: sistema-web\resources\views\historial\index.blade.php


2. INTERFAZ: PLANTILLAS BLADE
============================

Estructura general
------------------

sistema-web\resources\views\layouts\app.blade.php
Layout principal del personal. Carga cabecera, barra lateral, estilos, scripts y
el contenido de cada pantalla autenticada.

sistema-web\resources\views\layouts\navigation.blade.php
Menu lateral del personal. Aqui se agrupan sectores y subsectores y se controla
que opciones aparecen segun el rol.

sistema-web\resources\views\layouts\app-cliente.blade.php
Layout independiente del portal de consumidores. Evita mostrar herramientas
administrativas al cliente.

sistema-web\resources\views\layouts\guest.blade.php
Estructura para login, recuperacion de contrasena y paginas sin sesion.

Dashboard
---------

sistema-web\resources\views\home.blade.php
Dashboard operativo. Presenta solo menus efectivos por fecha y horario, platos,
precios y porciones restantes. Abre el registro rapido de consumo.

sistema-web\resources\views\consumos\modal-registro-rapido.blade.php
Modal del dashboard. Permite crear un pedido con varios platos del menu actual,
limita cantidades al stock disponible y selecciona opcionalmente un consumidor.

Menus del dia
-------------

platos y cantidades iniciales.

sistema-web\resources\views\menus-dia\edit.blade.php

13. REGLAS PARA MODIFICAR SIN ROMPER EL SISTEMA
===============================================

1. Una pantalla Blade no debe calcular saldos por su cuenta. Usar los metodos de
   Consumo y Consumidor para que Pagos y Consumidores siempre coincidan.

2. El dashboard no debe cargar recetas sueltas. Debe usar las recetas asociadas
   al MenuDia efectivo y su cantidad pivot disponible.

3. Toda venta desde el dashboard crea Venta. Si hay consumidor tambien crea
   Consumo pendiente y posteriormente los pagos se aplican a ese consumo.

4. Nunca descontar porciones sin bloquear el MenuDia en una transaccion. Esto
   evita vender dos veces la ultima porcion.

5. visible_para_clientes es permiso de publicacion. La visibilidad real tambien
   depende de activo, fecha y horario.

6. Toda operacion relevante debe llamar HistorialHelper::registrar con sector,
   accion y un detalle legible, evitando IDs sin nombres.

7. Los cambios de esquema se hacen con una migracion nueva. No editar una
   migracion ya ejecutada para corregir una base existente.

8. Despues de cambiar PHP o Blade ejecutar:
   php artisan view:clear
   php artisan test

9. Despues de cambiar JavaScript/CSS compilado ejecutar:
   npm run build

10. No editar archivos de storage/framework/views: son copias compiladas y se
    reemplazan automaticamente.
